# TBSI 时序令牌实验完整分析报告

## 零、核心发现

**历时 2 天、8 组实验、28 次训练测试。所有时序令牌变体均未产生正向增益。**

| 实验 | AUC | 与基线差 | 信号来源 |
|------|-----|---------|---------|
| DA基线(无令牌) | 55.83 | — | — |
| 前置v2 4ep | 55.32 | -0.51 | 4.73M随机参数=噪声 |
| 前置v2 15ep | 54.65 | -1.18 | 噪声累积 |
| 后置bn64无帧对 | 54.58 | -1.25 | 噪声下限 |
| 后置bn64+帧对 | 54.94 | -0.89 | 帧对修复分布偏移 |
| **后置bn128+帧对** | **55.40** | **-0.43** | **97%来自特征自注意力** |
| 纯交叉v3 | 48.97 | -6.86 | 信号饥饿 |
| EMA+交叉v4 | 未完成(~50预估) | — | 令牌坍缩+信号饥饿 |

## 一、全部设计的信号路径数学分析

### 1.1 v2 bn128（唯一"有效"版本 — 55.40）

**代码路径**（`post_fusion_token.py v2`）：

```
输入: fused_feat (B, 256, 768)  ← DA融合输出
  → Linear(768→128)              # 瓶颈压缩: 参数量 768×128+128 = 98,432
  → concat([feat(128), tokens(4)]) → (B, 260, 128)
  → norm + 自注意力               # 零参注意力, 无QKV投影
  → Linear(128→128)              # 输出投影: 128×128+128 = 16,512
  → split: feat(前256) + tok_raw(后4)
  → Linear(128→768)              # 恢复维度: 128×768+768 = 99,072
  → residual + fused_feat         # rs=0.1
  → MLP(768→96→768)              # 768×96+96 + 96×768+768 = 147,744
```

**参数量合计**: 98K + 16.5K + 99K + 147.7K + tokens(4×128) + norms ≈ **365K**

**注意力矩阵信号分解**：

```
注意力形状: (B, 4 heads, 260 tokens, 260 tokens)
             = 67,600 个注意力权重/head
             = 270,400 个权重/样本

┌──────────────┬──────────────┐
│              │  特征 (256)  │  令牌 (4)  │
├──────────────┼──────────────┼──────────────┤
│ 特征查询(256) │  65,536      │   1,024     │ ← 97.2% → ← 1.5%
│ 令牌查询(4)   │   1,024      │      16     │
├──────────────┼──────────────┼──────────────┤
│              │  = 66,560    │  = 1,040    │
└──────────────┴──────────────┴──────────────┘
总计: 67,600

特征↔特征: 65,536 (97.2%)  ← 真正的信号来源
特征↔令牌:  1,024 (1.5%)   ← 极微
令牌↔令牌:     16 (0.02%)  ← 可忽略
```

**结论**: 55.40 中 97.2% 的注意力信号来自特征自注意力，**令牌贡献 < 1.5%**。

### 1.2 后置迁移对DA融合的保护（为什么从 -1.32 到 -1.25）

```python
# 前置版本（损伤A: 差分毒化）
enc_rgb = backbone_output[:, 64:320, :]    # 原始backboneRGB特征
enc_tir = backbone_output[:, 320:, :]      # 原始backboneTIR特征
enc_rgb, enc_tir = temporal_token(enc_rgb, enc_tir)  # ← 4.73M QKV参数同时修改RGB和TIR
# DA融合接收被污染的特征
da_fused = da_fusion(enc_rgb, enc_tir)     # MADC: |enc_rgb|/|enc_tir|幅度比失真
                                             # CSR:  (enc_rgb - enc_tir)差分信号破坏

# 后置版本（修复A）
fused_feat = da_fusion(enc_rgb, enc_tir)   # DA融合在特征纯净时完成 ← ✅
fused_feat = post_fusion_token(fused_feat) # 融合后加时序 ← 不伤差分
```

**损伤A量化**: 前置15ep(54.65) vs 后置bn64无帧对(54.58) ≈ **-0.07**。后置本身收益小，因为在54.6噪声下限附近。

### 1.3 帧对训练的效果（+0.36，从54.58→54.94）

```python
# 训练时 forward():
if prev_search is not None and training:
    with torch.no_grad():
        # t-1帧: 走完整backbone+DA融合, 更新令牌
        fused_prev = self.forward_fusion_only(backbone(prev_search))
        _, tokens_updated = self.post_fusion_block(fused_prev, None)
    
    # t帧: 使用更新后的令牌
    fused_curr = self.forward_fusion_only(backbone(search))
    out = self.forward_head(feat_curr, prev_tokens=tokens_updated)
    # 关键: prev_tokens 不再是 init_tokens(固定参数)
    #       而是 tok_out_{t-1}(含真实前一帧信息)
```

**修复的原理**: 训练时令牌输入从固定 `init_tokens` 变为包含真实帧信息的动态变量，QKV投影的输入分布在训练和推理之间对齐。

**为什么收益只有 +0.36**: 因为v2的QKV投影在 bottleneck=64 的128维子空间做注意力，注意力图的主要信号仍然来自特征↔特征交互（97%）。令牌输入分布变化只影响1.5%的注意力权重。

### 1.4 bn64→bn128 的效果（+0.46，从54.94→55.40）

```python
# bn64:  Linear(768→64)    损失 99.5% 的维度（768→128→768 的瓶颈）
          feat_bot = input_proj(fused_feat)  # 768→64: 压缩比 12:1
# 注意力在64维空间运算:
          q @ k.T → (64/4)^(-0.5) = 4.0  # 注意力尺度过大, softmax接近one-hot
          
# bn128: Linear(768→128)   损失 83% 的维度  
          feat_bot = input_proj(fused_feat)  # 768→128: 压缩比 6:1
# 注意力在128维空间运算:
          q @ k.T → (128/4)^(-0.5) = 5.66  # 注意力尺度更合理
```

**信息论视角**: 768维特征的信息熵 ~ H[x]，64维投影保留的信息 I(x; x_proj) ≤ 64/768 × H[x] ≈ 8.3%H[x]。128维 ≈ 16.7%H[x]。bn128 保留的信息量是 bn64 的 2 倍，但代价是从 130K → 230K 参数。

## 二、v3 和 v4 的失败路径

### 2.1 v3 纯交叉注意力（48.97）

**设计假设**: "特征自注意力是冗余的，因为主干ViT已经做了12层。"

**致命错误**: 主干ViT的12层自注意力在768维空间做（12 × 768 × 768），**后置模块的瓶颈自注意力在128维空间做（1 × 128 × 128）**。两者尺度不同、位置不同、功能不同：
- 主干自注意力: 建模模板+搜索区域的联合空间关系（768维）
- 后置瓶颈自注意力: 在128维空间做DA融合后的特征精调

**信号量对比**:
```
v2 bn128 自注意力:
  attn = q @ k.T → (B, 4, 260, 260)  
  每个位置i的注意力输出:
    out[i] = Σ_j attn[i][j] × v[j]   (j=1..260)
    每个位置可以从259个其他位置获取信息
    信息容量: O(260²)

v3 纯交叉注意力:
  attn = q @ k.T → (B, 4, 256, 4)
  每个位置i的注意力输出:
    out[i] = Σ_k attn[i][k] × token[k]   (k=1..4)
    每个位置只能从4个token获取信息
    信息容量: O(256×4) = O(1024)

信息比: 67,600 / 1,024 ≈ 66:1
```

**数学本质**: `out[i] = α_i1·t1 + α_i2·t2 + α_i3·t3 + α_i4·t4`

这是256个位置在4个基向量张成的子空间上的投影。256个位置只能表达4种"原型"的不同混合比例，**空间结构信息完全丢失**。相当于对特征图做"4均值池化"再插值回去。

### 2.2 v4 EMA + 交叉检索（预估~50）

**设计假设**: "显式存特征比隐式学习更稳定。"

**EMA 更新的数学演化**:
```
Let:
  init_tokens = [t₁⁰, t₂⁰, t₃⁰, t₄⁰]  ← 第一帧特征采样  (4个不同向量)
  α = σ(log_update_rate) ≈ 0.1          ← EMA更新率
  
第1帧:
  t_k¹ = α·g¹ + (1-α)·t_k⁰            ← g¹ = mean(feat¹), 全局特征
    = 0.1·g¹ + 0.9·t_k⁰               ← 4个token向g¹靠近

第2帧:
  t_k² = α·g² + (1-α)·t_k¹
    = 0.1·g² + 0.9·(0.1·g¹ + 0.9·t_k⁰)
    = 0.1·g² + 0.09·g¹ + 0.81·t_k⁰

第N帧（几何级数）:
  t_k^N = α·Σ_{i=1}^{N} (1-α)^{N-i}·gⁱ + (1-α)^N·t_k⁰

当 N→∞:  t_k^N → α·Σ_{i=1}^{∞} (1-α)^{i-1}·g^{N-i+1}  ← tokens只与历史全局均值相关
```

**关键**: 不同帧的 `gⁱ = mean(featⁱ)` 都是256个空间位置的全局均值，差异小。因此:

```
N=5:   t_k^5 = 0.41·g_avg + 0.59·t_k⁰
N=10:  t_k^5 = 0.65·g_avg + 0.35·t_k⁰
N=20:  t_k^20 = 0.88·g_avg + 0.12·t_k⁰

4个token的差异矩阵:
  ||t₁^N - t₂^N|| = (1-α)^N·||t₁⁰ - t₂⁰||
  → 指数衰减到 0
```

**N=20时, 4个token几乎一致 → 交叉注意力输出各位置相等 → 等同于对整图加全局偏置 → 无效**

## 四、最终根因

### 4.1 核心矛盾

时序令牌在LasHeR上无法涨点的根因有三层:

**第一层（表象）**: 训练监督信号不足。
- 梯度路径: GIoU/L1 loss → 预测头 → 融合特征 → residual_scale(0.1) → attention → tok_params
- 到达tok_params的梯度 ≈ 主梯度的 0.1%~1%
- 3ep微调相当于只让令牌参数看到 ~45 次梯度更新

**第二层（结构）**: 令牌容量与任务需求不匹配。
- 4个768维向量存储"历史记忆" → 4×768 = 3072 个值
- 每次查询需要从4个值中检索信息 → 极限信息容量 = 4 × 768 = 3072 bit
- LasHeR序列平均200帧, 每帧特征768×256 ≈ 196,608 维 → 令牌能存储的信息不到输入的 1.6%

**第三层（本质）**: 特征自注意力已经实现了令牌意图的功能。
- 在瓶颈自注意力中, 每个位置从其他255个位置获取信息
- 如果特征是平滑的（相邻patch相似）, 自注意力等价于局部平滑滤波
- 令牌相当于增加4个额外的"虚拟patch", 但255个真实patch已经提供了足够的上下文
- **增加4个额外位置不会改变注意力图的分布特征, 但增加了4×260=1040个需要学习的注意力权重**

### 4.2 数学不等式

```
AUC_gain(tokens) ≤ η × C(tokens) / C(total)

其中:
  η = 时序信息的可利用率 (在LasHeR短序列上 < 0.3)
  C(tokens) = token在注意力中的信息容量 (1,024 交互对)
  C(total) = 注意力总信息容量 (67,600 交互对)
  
AUC_gain ≤ 0.3 × 1,024/67,600 ≈ 0.3 × 1.5% ≈ 0.45%

实际观察到的最大增益 (从噪声下限54.58到最优55.40):
  gain = 55.40 - 54.58 = 0.82 (其中约0.46来自bn64→bn128的瓶颈缓解)
  令牌自身贡献 ≈ 0.82 - 0.46 ≈ 0.36, 且其中大部分来自帧对训练对齐分布
  
令牌净贡献 ≈ 0.36 - (分布式偏移造成的4ep→15ep的-0.67) ≈ -0.31
```

**结论: 在LasHeR上, 时序令牌的理论最大正向收益 < 0.5% AUC。在实际训练中, 这个微弱信号完全被参数噪声淹没。任何可学习时序令牌在这个数据集上都无法实现可证明的正向收益。**
