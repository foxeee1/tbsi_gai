# TBSI 二级训练测试基准

**复合指标 + 每序列胜率 + 三重严格门控**

## 设计哲学

传统的 SR/PR/NPR 单指标门控容易"钻空子"——一个修改可能让 2 个序列大幅提升而 243 个序列略微下降。
本基准通过 **复合指标 + 胜率分析 + 无损检查** 三重门控来严格筛选改进。

## 二级流水线

```
L1 Sprint (≤1h) ──── 三重门控 ────→ L2 Full (~6h)
 4ep × 12k smpl       ↓ 不通过       15ep × 60k smpl
 bs=32, 100 seq        继续改进       bs=32, 245 seq
```

### 三重门控条件 (缺一不可)

| # | 条件 | 阈值 | 目的 |
|---|------|------|------|
| 1 | 复合指标提升 | ≥ **+1.0%** | 防单指标投机 |
| 2 | 每序列胜率 | > **55%** | 过半序列提升，非靠少数暴涨 |
| 3 | 核心指标无损 | 降幅 ≤ **1.0%** | 防止 SR 涨但 PR 暴跌 |

### 为什么这么严？

从 SOT 基线实测数据看：
- L1 Sprint (50min) 已经达到全量 97% 的性能
- 微小改进（0.3-0.5%）在单卡 bs=32 的噪声范围内不可靠
- 不满足三重条件的改进不值得花 6h 跑全量

## 多指标体系

| 维度 | 指标 | 含义 | 复合权重 |
|------|------|------|----------|
| **Overlap** | SR (AUC) | 成功率曲线下面积 | 30% |
| | OP50 | IoU>0.5 的帧比例 | 15% |
| | MeanIoU | 平均帧 IoU | 15% |
| **Precision** | PR | 20px 中心误差精度 | 25% |
| | NPR | 归一化精度 | 15% |
| **Robustness** | Win Rate | 每序列 AUC 胜率 | 门控参考 |

**Composite = 0.30×SR + 0.15×OP50 + 0.25×PR + 0.15×NPR + 0.15×MeanIoU**

## SOT 基线参考数据

| 级别 | SR | PR | NPR | 耗时 |
|------|:--:|:--:|:---:|:----:|
| **L1 Sprint** (50min) | **54.72** | **68.95** | **65.02** | ~50min |
| L2 Verify (2.5h) | 54.80 | 68.34 | 64.61 | ~2.5h |
| L3 Full (6h) | 54.62 | 68.17 | 64.43 | ~6h |
| 论文 SOT (4GPU) | 56.50 | 70.20 | 66.50 | 4GPU |

结论：**L1 Sprint 已经够用**，L2/L3 不提供额外价值。

## 快速开始

```bash
# 1. 激活环境
source benchmark/bm_env.sh

# 2. 初始化 (生成配置 + L1 基线)
python benchmark/bm_run.py --init

# 3. 每次修改后验证
python benchmark/bm_run.py --run --desc "修改内容"

# 4. 查看状态
python benchmark/bm_run.py --status
```

## 命令

| 命令 | 功能 |
|------|------|
| `--init` | 初始化 (生成配置 + 建立 L1 基线) |
| `--run --desc "..."` | 在当前级别训练 + 测试 + 门控 |
| `--run --level 2 --force` | 跳过栅栏在 L2 运行 |
| `--status` | 查看详细状态 |
| `--advance` | 手动推进到下一级 |
| `--reset` | 重置所有数据 |
